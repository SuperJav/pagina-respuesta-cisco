Capítulo 5 SIC: Lista de control de acceso y configuración de SNMP


1.No utilice el botón Atrás del navegador ni cierre ni vuelva a cargar las ventanas de examen durante el examen.
2.No cierre Packet Tracer cuando haya terminado. Se cerrará automáticamente.
3.Haga clic en el botón Enviar evaluación para enviar su trabajo.


Introducción


En esta práctica de la Evaluación basada en habilidades de Packet Tracer, usted:


  • Configurar cadenas de comunidad SNMP.
  • Configure las ACL IPv4 estándar y extendidas para filtrar el tráfico de red.
  • Configure una ACL de IPv6 para filtrar el tráfico de red.

Tabla de direccionamiento



Paso 1: configure las cadenas de la comunidad SNMP en el enrutador HQ.


a. Configure una cadena de comunidad de solo lectura SNMP hq-monitor .
segundo. Configure una cadena de comunidad SNMP de lectura / escritura hq-inside .


Paso 2: Configure una ACL para NAT en el enrutador HQ.


a. Configure la lista de acceso estándar numerada 1 para permitir NAT para hosts en la red 192.168.10.0 / 24.


Paso 3: configure una ACL estándar para restringir el acceso remoto al enrutador HQ.


a. Configure una ACL estándar numerada 12 para restringir el acceso remoto a HQ.

  • Permitir que solo el HQ-Admin-PC acceda al enrutador HQ de forma remota a través de VTY.
  • Todas las demás conexiones remotas deberían fallar.

Paso 4: configure dos ACL extendidas para restringir el acceso a la operación SNMP en el enrutador HQ.

a. Configure una ACL extendida llamada SNMPACCESS.

  • La operación SNMP ejecuta UDP en el puerto 161.
  • Permitir que solo el HQ-Admin-PC acceda al enrutador HQ para la conexión SNMP.
  • Las conexiones SNMP de otros hosts en la LAN HQ deben fallar.
  • Permitir todo el tráfico IP.
  • Aplique esta ACL en el enrutador HQ, interfaz G0 / 0.

segundo. Configure una ACL extendida llamada SNMPDENY.

  • Denegar a los hosts que realicen conexiones a SNMP en el enrutador HQ.
  • Permitir todo el tráfico IP.
  • Aplique esta ACL en el enrutador HQ, interfaz G0 / 1.

Paso 5: configure una ACL extendida para restringir el acceso a la HQ LAN desde Internet.

a. Configure una ACL IPv4 extendida llamada INTOHQ.

  • Permitir que los hosts de Internet accedan a la Sede de DNS de HQ. Debe haber dos ACE, una para TCP y la otra UDP. Ambos utilizan el puerto 53.
  • Permitir que los hosts de Internet accedan a HQ Web Svr. Solo se necesita el puerto 80.
  • Permitir que pase el tráfico TCP de retorno desde Internet que se inició desde los hosts en las redes de HQ (con la palabra clave establecida ).
  • Aplicar la ACL a la interfaz HQ S0 / 0/0.

Paso 6: configure una ACL extendida para restringir el acceso a la red DMZ.

a. Configure una ACL IPv4 extendida llamada IN-DMZ.

  • Permitir que los hosts accedan a HQ DNS Svr. Debe haber dos ACE, una para TCP y la otra UDP. Ambos utilizan el puerto 53.
  • Permitir que cualquier host acceda a HQ Web Svr. Solo se necesita el puerto 80.
  • Permitir que solo el HQ-Admin-PC tenga acceso de FTP al servidor web de HQ. Debería haber dos ACE, para los puertos 20 y 21.
  • Aplicar la ACL a la interfaz HQ G0 / 1.

Paso 7: configure una ACL de IPv6 para restringir el acceso a la red DMZ desde la red de sucursal. ( Tenga en cuenta que el orden de las declaraciones de ACL es significativo solo debido a la necesidad de puntuación en Packet Tracer ) .

a. Configure una ACL de IPv6 llamada DMZFTP.

  • Denegar cualquier host en la red de Sucursal para acceder a la operación SNMP del enrutador HQ.
  • Permitir que solo BR-Admin-PC tenga acceso FTP al servidor web de HQ. Debería haber dos ACE, para los puertos 20 y 21.
  • Permitir que los hosts en la Red de Sucursales accedan al servidor web de HQ. Solo se necesita el puerto 80.
  • Aplique la ACL a la interfaz S0 / 0/1 del enrutador HQ.

Paso 8: Pruebas de conectividad

a. HQ-Admin-PC puede acceder al servicio FTP en HQ Web Svr.
segundo. HQ-User-PC no puede acceder al servicio FTP en HQ Web Svr.
do. BR-Admin-PC puede acceder al servicio FTP en HQ Web Svr con su dirección IPv6.
re. BR-User-PC no puede acceder al servicio FTP en HQ Web Svr con su dirección IPv6.
mi. HQ-Admin-PC, HQ-User-PC y External-PC pueden acceder a HQ Web Svr con URL www.hq.com
F. HQ-Admin-PC, HQ-User-PC y External-PC pueden acceder a External Web Srv con URL www.ext-web.com
sol. BR-Admin-PC y BR-User-PC pueden acceder a HQ Web Svr con su dirección IPv6
h. BR-Admin-PC y BR-User-PC pueden acceder a External Web Srv con URL www.ext-web.com

Script de configuración para el enrutador: HQ / Main / Central (100%)

ES
conf ter
comunidad snmp-server hq-monitor ro
comunidad snmp-server hq-inside rw
Permiso de la lista de acceso 1 192.168.10.0 0.0.0.255
lista de acceso 12 permiso de host 192.168.10.5
línea vty 0 1
clase de acceso 12 en
salida


ip access-list extendido SNMPACCESS
Permitir udp host 192.168.10.5 host 192.168.10.1 eq 161
negar a udp cualquier host 192.168.10.1 eq 161
Permitir ip cualquiera cualquiera
interfaz g0 /
ip access-group SNMPACCESS en
salida

ip-lista de acceso extendida SNMPDENY
negar a udp cualquier host 172.16.10.1 eq 161
Permitir ip cualquiera cualquiera
interfaz g0 / 1
ip access-group SNMPDENY EN
salida
ip access-list extendido INTOHQ
Permitir tcp cualquier host 172.16.10.5 eq 53
Permitir udp cualquier host 172.16.10.5 eq 53
permite tcp cualquier host 172.16.10.10 eq 80
Permitir tcp cualquiera establecido
interfaz s0 / 0/0
ip access-group INTOHQ IN
salida


ip lista de acceso extendida en DMZ
Permitir udp cualquier host 172.16.10.5 eq 53
Permitir tcp cualquier host 172.16.10.5 eq 53
permite tcp cualquier host 172.16.10.10 eq 80
Permitir host tcp 192.168.10.5 host 172.16.10.10 eq 20
Permitir host tcp 192.168.10.5 host 172.16.10.10 eq 21
interfaz g0 / 1
ip access-group IN-DMZ out
salida
ipv6 lista de acceso DMZFTP
niega a udp cualquier host 2001: DB8: ABCD: E :: 1 eq 161
Permitir tcp host 2001: DB8: ABCD: B :: 5 host 2001: DB8: ABCD: A :: 10 eq 20
permiso tcp host 2001: DB8: ABCD: B :: 5 host 2001: DB8: ABCD: A :: 10 eq 21
permiso tcp 2001: DB8: ABCD: B :: / 64 host 2001: DB8: ABCD: A :: 10 eq 80
int s0 / 0/1
ipv6 traffic-filter DMZFTP IN

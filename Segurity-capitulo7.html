6.¿En qué situación se utiliza un algoritmo de clave asimétrica?

  • Dos routers Cisco se autentican entre sí con CHAP.
  • Los datos del usuario se transmiten a través de la red después de establecer una VPN.

  • Un administrador de red se conecta a un enrutador Cisco con SSH. *
  • @Javi__Super

El protocolo SSH utiliza un algoritmo de clave asimétrica para autenticar a los usuarios y cifrar los datos transmitidos. El servidor SSH genera un par de claves públicas / privadas para las conexiones. El cifrado de archivos antes de guardarlos en un dispositivo de almacenamiento utiliza un algoritmo de clave simétrica porque la misma clave se utiliza para cifrar y descifrar archivos. La autenticación del enrutador con CHAP utiliza un algoritmo de clave simétrica. La clave está preconfigurada por el administrador de la red. Una VPN puede usar tanto una clave asimétrica como un algoritmo de cifrado simétrico. Por ejemplo, en una implementación IPSec VPN, la transmisión de datos utiliza un secreto compartido (generado con un algoritmo de clave asimétrica) con un algoritmo de cifrado simétrico utilizado para el rendimiento.

7.¿Por qué a menudo se prefiere el algoritmo 3DES sobre el algoritmo AES?

  • Se confía más en 3DES porque se ha demostrado que es seguro durante un período más largo que AES. *
  • AES es más caro de implementar que 3DES.
  • 3DES se desempeña mejor en entornos de alto rendimiento y baja latencia que AES.
  • Los principales proveedores de equipos de redes, como Cisco, aún no han adoptado AES.
  • @Javi__Super

A pesar de sus ventajas, AES es un algoritmo relativamente joven. Una regla importante de la criptografía es que un algoritmo maduro siempre es más confiable. Por lo tanto, 3DES es una opción más confiable en términos de fuerza, ya que ha sido probado y analizado durante 35 años. AES se puede utilizar en entornos de alto rendimiento y baja latencia, especialmente cuando 3DES no puede manejar los requisitos de rendimiento o latencia. AES está disponible en varios dispositivos VPN de Cisco como una transformación de cifrado.

Diffie-Helman no es un mecanismo de cifrado y generalmente no se utiliza para cifrar datos. En su lugar, es un método para intercambiar de forma segura las claves utilizadas para cifrar los datos.

La criptología es la ciencia de hacer y romper códigos secretos. Hay dos disciplinas separadas en criptología, criptografía y criptoanálisis. La criptografía es el desarrollo y uso de códigos. El criptoanálisis es la ruptura de esos códigos secretos (encriptados).

10.¿Cuántos bits utiliza el Estándar de cifrado de datos (DES) para el cifrado de datos?

  • 40 bits
  • 56 bits *
  • 64 bits
  • 72 bits
  • @Javi__Super

DES utiliza una clave de longitud fija. La clave tiene una longitud de 64 bits, pero solo se utilizan 56 bits para el cifrado. Los 8 bits restantes se utilizan para la paridad. Una clave de cifrado DES siempre tiene una longitud de 56 bits. Cuando se usa DES con un cifrado más débil de una clave de 40 bits, la clave de cifrado es de 40 bits secretos y 16 bits conocidos, lo que hace que la longitud de la clave sea de 56 bits.

14.¿Cómo se defienden los criptógrafos modernos contra los ataques de fuerza bruta?

  • Utilice el análisis estadístico para eliminar las claves de cifrado más comunes.
  • Use un espacio de teclas lo suficientemente grande como para que se requiera demasiado dinero y demasiado tiempo para realizar un ataque exitoso. *
  • Utilice un algoritmo que requiera que el atacante tenga tanto texto cifrado como texto sin formato para realizar un ataque exitoso.
  • Use el análisis de frecuencia para asegurarse de que las letras más populares utilizadas en el idioma no se usen en el mensaje cifrado.
  • @Javi__Super

En un ataque de fuerza bruta, un atacante intenta cada clave posible con el algoritmo de descifrado sabiendo que eventualmente uno de ellos funcionará. Para defenderse de los ataques de fuerza bruta, los criptógrafos modernos tienen como objetivo tener un espacio de teclas (un conjunto de todas las claves posibles) lo suficientemente grande como para que requiera demasiado dinero y demasiado tiempo para llevar a cabo un ataque de fuerza bruta. Una política de seguridad que requiere que las contraseñas se cambien en un intervalo predefinido defiende aún más contra los ataques de fuerza bruta. La idea es que las contraseñas se hayan cambiado antes de que un atacante agote el espacio de claves.

16.¿Qué dos números no secretos se acuerdan inicialmente cuando se usa el algoritmo Diffie-Hellman? (Escoge dos.)

  • coeficiente binomial
  • generador *
  • curva elíptica invariante
  • módulo primario *
  • @Javi__Super

DH es un algoritmo matemático que permite que dos hosts generen un secreto compartido idéntico en ambos sistemas sin haberse comunicado antes. Para iniciar un intercambio de DH, ambos hosts deben acordar dos números no secretos. El primer número es un número base, también llamado generador. El segundo número es un número primo que se utiliza como módulo. Estos números suelen ser públicos y se eligen de una tabla de valores conocidos.

17.¿Qué tipo de algoritmo de cifrado usa la misma clave para cifrar y descifrar datos?

  • Diffie-Hellman
  • Secreto compartido *
  • Llave pública
  • Asimétrico
  • @Javi__Super

Los algoritmos de cifrado simétrico usan la misma clave (también llamada secreto compartido) para cifrar y descifrar los datos. En contraste, los algoritmos de cifrado asimétrico (también llamados clave pública) usan un par de claves, una para el cifrado y otra para el descifrado.

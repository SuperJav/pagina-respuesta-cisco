  • Utiliza la dirección IP de origen en el paquete entrante.

3.¿Cuáles son dos beneficios de NAT? (Escoge dos.)

  • Guarda las direcciones IP públicas. *
  • Agrega un grado de privacidad y seguridad a una red. *
  • Aumenta el rendimiento de enrutamiento.
  • Hace que la tunelización con IPsec sea menos complicada.
  • @javi__super

4.¿Qué es una desventaja de NAT?

  • No hay direccionamiento de extremo a extremo. *
  • El enrutador no necesita alterar la suma de comprobación de los paquetes IPv4.
  • Los costos de redireccionar los hosts pueden ser significativos para una red dirigida públicamente.
  • Los hosts internos tienen que usar una única dirección IPv4 pública para la comunicación externa.
  • @javi__super

5.¿Cuál es una ventaja de implementar la tecnología NAT IPv4 para hosts internos en una organización?

  • proporciona flexibilidad en el diseño del esquema de direccionamiento IPv4 *
  • Aumenta el rendimiento de la transmisión de paquetes a Internet.
  • facilita el acceso a la red interna para hosts externos utilizando UDP
  • permite la implementación sencilla de aplicaciones que requieren una trazabilidad de extremo a extremo

15.¿Cuál es el propósito del reenvío de puertos?

  • El reenvío de puertos permite a un usuario externo acceder a un servicio en una dirección IPv4 privada que se encuentra dentro de una LAN. *
  • El reenvío de puertos permite a un usuario interno acceder a un servicio en una dirección IPv4 pública que se encuentra fuera de una LAN.
  • El reenvío de puertos permite a los usuarios acceder a servidores en Internet que no están utilizando números de puerto estándar.
  • El reenvío de puertos permite traducir direcciones IP locales a direcciones locales externas.

3.¿Qué información debe rastrear un IPS para detectar ataques que coincidan con una firma compuesta?

  • El número total de paquetes en el ataque.
  • El periodo de ataque utilizado por el atacante.
  • El ancho de banda de red consumido por todos los paquetes.
  • el estado de los paquetes relacionados con el ataque *
  • @Javi__Super

Una firma compuesta se llama una firma con estado. Identifica una secuencia de operaciones distribuidas en varios hosts durante un período de tiempo arbitrario. Debido a que este tipo de ataque involucra múltiples paquetes, un sensor IPS debe mantener la información de estado. Sin embargo, un sensor IPS no puede mantener la información de estado de forma indefinida. Una firma compuesta se configura con un período de tiempo para mantener el estado del ataque específico cuando se detecta por primera vez. Por lo tanto, es posible que un IPS no pueda mantener toda la información relacionada con un ataque, como el número total de paquetes, la duración total del tiempo de ataque y la cantidad de ancho de banda consumido por el ataque.

4.¿Qué es una desventaja de un mecanismo de detección basado en patrones?

  • El patrón de tráfico de red normal debe ser perfilado primero.
  • No puede detectar ataques desconocidos. *
  • Es difícil de implementar en una red grande.
  • Su configuración es compleja.
  • @Javi__Super

Un IDS / IPS con detección basada en patrones, también conocida como detección basada en firmas, compara el tráfico de red con una base de datos de ataques conocidos (archivos de firmas) y activa una alarma o evita la comunicación si se encuentra una coincidencia. Las firmas deben ser creadas primero. Por lo tanto, este tipo de detección de intrusiones no puede detectar ataques desconocidos. Es fácil de configurar y desplegar. Su funcionamiento no depende de la información del comportamiento normal de la red (o línea de base).

8.Un analista de sistemas está configurando y ajustando un dispositivo IPS recientemente implementado. Al examinar el registro de alarmas de IPS, el analista advierte que el IPS no genera alarmas para unos pocos paquetes de ataque conocidos. ¿Qué término describe la falta de alarmas por parte del IPS?

  • verdadero negativo
  • falso positivo
  • falso negativo *
  • verdadero positivo
  • @Javi__Super

Las alarmas generadas por un IPS se pueden clasificar en 4 tipos:
Un falso positivo ocurre cuando un IPS genera una alarma en el tráfico normal de usuarios que no debería haber activado una alarma. Un falso negativo ocurre cuando un IPS no genera una alarma después de procesar el tráfico de ataque que el IPS está configurado para detectar. Un verdadero positivo ocurre cuando un IPS genera una alarma en respuesta al tráfico de ataque conocido. Un verdadero negativo ocurre cuando el tráfico normal de la red no genera una alarma.

9.Un especialista en seguridad configura un IPS para que genere una alerta cuando se detecte un ataque por primera vez. Las alertas para la detección posterior del mismo ataque se suprimen durante un período de tiempo predefinido. Se generará otra alerta al final del período que indica el número del ataque detectado. ¿Qué mecanismo de monitoreo de alertas IPS está configurado?

  • alerta compuesta
  • alerta atómica
  • alerta de correlación
  • alerta de resumen *
  • @Javi__Super

Las alertas generadas por un IPS se deben supervisar de cerca para garantizar que se tomen las medidas adecuadas contra los ataques malintencionados. Las soluciones IPS incorporan dos tipos de alertas, alertas atómicas y alertas de resumen. Las alertas atómicas se generan cada vez que se desencadena una firma. Una alerta de resumen es una alerta única que indica varias apariciones de la misma firma desde la misma dirección o puerto de origen. Con un cambio de resumen, la primera detección del ataque desencadena una alerta normal. La detección posterior del mismo ataque se cuenta hasta el final del intervalo de resumen de la firma. Cuando el tiempo especificado por el intervalo de resumen ha transcurrido, se envía una alarma de resumen, que indica el número de alarmas que se produjeron durante el intervalo de tiempo.

10.¿Qué afirmación es cierta acerca de una alerta atómica generada por un IPS?

  • Es una única alerta enviada para múltiples ocurrencias de la misma firma.
  • Es una alerta que se genera cada vez que se encuentra una firma específica. *
  • Es una alarma normal y una alarma de resumen que se envían simultáneamente a intervalos establecidos.
  • @Javi__Super

Los dos mecanismos principales de generación de alertas para dispositivos IDS / IPS son alertas atómicas y de resumen. Las alertas atómicas se generan cada vez que se desencadena una firma. Con una alerta de resumen, se genera una sola alerta atómica para la primera detección de un ataque. Luego, las alarmas duplicadas se cuentan, pero no se envían, durante un período de tiempo específico. Cuando alcanza el período de tiempo especificado, se envía una alerta que indica el número de alarmas que se produjeron durante el intervalo de tiempo.

18.¿Cuáles son las dos desventajas de usar un IDS? (Escoge dos.)

  • El IDS no tiene impacto en el tráfico.
